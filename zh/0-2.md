# 变量和常量

每一个有用的程序都需要在某个时候存储数据，而在Swift中有两种存储数据的方式：变量和常量。变量是随时可以更改其值的数据存储；而常量是只要设置一次，就永远不能更改的数据存储。因此，变量的值可以变化，常量的值是常量。很简单，对吗？

拥有这两种选项似乎都毫无意义，毕竟，您可以创建一个变量，然后永远不改变它——为什么需要将其设置为常量？事实上，很多程序员都很震惊！在编程方面都不会太完美，而且我们会犯错误。

把常量和变量分开的优点之一是 Xcode 会告诉我们是否犯了错误。如果我们说“把这个日期设置成常量，因为我知道它永远不会改变”，那么10行之后试图改变它，Xcode将无法构建我们的应用程序。

常量也很重要，因为它们让 Xcode 对构建应用程序的方式做出决定。如果它知道一个值永远不会改变，它能够应用优化使代码运行得更快。

在 Swift 中，使用 `var` 关键字定义变量，如下所示：

    var name = "Tim McGraw"

让我们把它放进 playground，这样你就可以开始得到反馈了。除了 `import UIKit` 行之外，删除其中的所有内容（这是苹果的核心iOS框架中需要使用的部分），然后添加该变量。你应该看看下面的图片。

![在 Xcode playgrounds, 在左边键入代码，然后在右边看到结果。](0-2.png)

因为这是一个变量，所以您可以随时更改它，但是不应该每次都使用 `var` 关键字——这只在声明新变量时使用。试着写这个：

    var name = "Tim McGraw"
    name = "Romeo"

因此，第一行创建 `name` 变量并给它一个初始值，然后第二行更新 `name` 变量，所以它的值现在是“Romeo”。你会看到两个值打印在 playground 的结果区域。

那么，如果我们把它变成常量而不是变量呢？嗯，常量使用的是 `let` 关键字而不是 `var`，所以您可以将第一行代码改为 `let name` 而不是 `var name`，如下所示：

    import UIKit
    let name = "Tim McGraw"
    name = "Romeo"

但是现在有一个问题：Xcode 在第三行旁边显示一个红色警告符号，它应该在 `name` 下面画一个下划线。如果单击红色警告符号，Xcode 将告诉您问题："Cannot assign to 'let' value 'name'"——这是Xcode所表示的“您试图更改一个常量，但是您无法这样做”。


![如果您试图在 Swift 中更改常量，Xcode 将拒绝构建应用程序。](0-3.png)

因此，常量是向 Swift 和您自己保证一个值不会改变的好方法，因为如果您试图更改它，Xcode 将拒绝运行。Swift 开发人员强烈倾向于尽可能使用常量，因为它使代码更容易理解。事实上，在最新版本的 Swift 中，如果你定义了一个变量，并且没有改变它Xcode 实际上会提示你。


**重要提示:** 变量和常量名称在代码中必须是唯一的。如果你尝试两次使用相同的变量名，你会得到一个错误，比如：

    var name = "Tim McGraw"
    var name = "Romeo"

如果 playground 在代码中发现错误，它会在红色框中标记警告，或者拒绝运行。您会知道后者是否发生了，因为结果窗格中的文本已经变为灰色，而不是通常的黑色。
